## MODIFIED Requirements

### Requirement: 手动 URL 补采触发
系统 MUST 提供手动补采入口，支持后台页面与 CLI 两种入口提交文章 URL，并要求显式传入平台编码触发采集任务。

#### Scenario: 通过后台页面提交合法 URL 列表
- **WHEN** 管理员在后台导入页选择平台并提交合法 URL 列表
- **THEN** 系统创建补采任务并进入统一任务流转

#### Scenario: 通过 CLI 提交合法文章 URL
- **WHEN** 运维通过 CLI 提交合法 URL 且指定平台
- **THEN** 系统立即创建补采任务并进入统一任务流转

### Requirement: 自动与手动任务统一纳管
系统 SHALL 将手动补采任务与自动采集任务纳入同一任务模型、状态机与日志体系，并在任务记录中保留平台编码。

#### Scenario: 查询任务执行明细
- **WHEN** 维护人员查看某次手动补采执行结果
- **THEN** 系统返回与自动任务一致的状态、错误信息、重试记录以及任务平台编码

### Requirement: 补采幂等控制
系统 MUST 对手动补采执行幂等校验，避免同一平台下同一文章被重复创建多个并发任务。

#### Scenario: 短时间重复提交同一平台同一 URL
- **WHEN** 维护人员连续提交同一平台下同一篇文章 URL
- **THEN** 系统仅保留一个有效执行任务并返回幂等提示
