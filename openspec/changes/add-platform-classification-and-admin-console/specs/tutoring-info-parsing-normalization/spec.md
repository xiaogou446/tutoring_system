## MODIFIED Requirements

### Requirement: 家教关键信息抽取
系统 MUST 从文章文本中抽取可检索字段（城市、区域、年级、科目、地址、时间安排、薪资、老师要求与发布时间），并按 `platformCode` 选择对应解析规则。

#### Scenario: 已注册平台执行字段抽取
- **WHEN** 任务携带已注册的平台编码
- **THEN** 系统按该平台解析规则输出结构化字段并与原文建立可追溯关联

### Requirement: 不规则文本与缺失字段兼容
系统 SHALL 支持非固定格式文本解析，对无法识别的字段留空而非中断整条记录入库。

#### Scenario: 不同平台字段模板不一致
- **WHEN** 两个平台文章字段模板不同且部分字段缺失
- **THEN** 系统在各自规则下保存可识别字段并将缺失字段置为空值

### Requirement: 原始片段回溯能力
系统 MUST 在结构化结果中保留字段对应原始片段，并保留来源平台编码以支持跨平台核对与规则迭代。

#### Scenario: 运营跨平台核对解析结果
- **WHEN** 运营人员查看结构化字段来源
- **THEN** 系统可返回对应原文片段、文章来源链接与平台编码
